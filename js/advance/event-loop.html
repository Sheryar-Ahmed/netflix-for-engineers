The 5 pieces, explained simply

    

      

        1 · Call Stack
        
Where code actually runs

        
A stack of function calls. The function on top runs to completion, then pops off. There's only one — that's what "single-threaded" means.

      

      

        2 · Web APIs
        
The browser's helpers

        
Timers, fetch, geolocation, DOM events… When you call these, the browser does the slow waiting off the stack, so your page never freezes.

      

      

        3 · Task Queue
        
Callbacks wait here

        
When a setTimeout or event finishes, its callback lines up here (also called the callback queue) — waiting for a free stack.

      

      

        4 · Microtask Queue
        
The VIP line ⭐

        
For Promise .then/.catch/.finally, code after await, and queueMicrotask. It has priority and is fully drained before any task runs.

      

      

        5 · Event Loop
        
The traffic cop

        
Loops forever: "Is the stack empty?" If yes → drain all microtasks, then take one task. Re-check microtasks after every task.

      

      

        ★ The golden rule
        
Order of execution

        
    
          
  • Synchronous code first (top to bottom)
    • 
          
  • then all microtasks (Promises)
    • 
          
  • then one task (setTimeout) → repeat
